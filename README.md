# DTCup 备考全链路智能刷题与资料聚合系统

## 📝 项目背景
本项目专为 **“大学生新一代信息通信科技大赛（DTCup）”** 备赛设计。

针对开源备赛资料分散、题目格式不一、缺乏高效交互式练习环境等痛点，本项目构建了一套从 **“非结构化资料”** 到 **“结构化题库”**，再到 **“本地交互式刷题”** 的全链路自动化系统，旨在提升备考效率与知识内化质量。

---

## 🛠️ 核心架构与工作流
本项目整合了多个开源仓库资源，并搭建了自动化的数据清洗与注入管线：

1.  **题源聚合 (Data Aggregation)**
    * 提取 `dtcup-2026-prep` 仓库中的高频考点、课程笔记和真题（Markdown 格式）。
    * 解析 `2025dtCup-Qbank` 仓库中的补充题库资源。

2.  **自动化数据清洗 (Data Pipeline)**
    * **正则提取**：使用 Python 脚本解析 Markdown 文本，自动识别题干、选项、正确答案与解析。
    * **结构化中转**：将提取的数据导出为 Excel 格式，便于进行人工去重、排版校对和图像路径映射。
    * **JSON 序列化**：通过 Python (`pandas`, `json`) 将清洗后的数据一键转化为标准 JSON 格式。

3.  **本地刷题引擎 (Quiz Engine)**
    * 将生成的 `questions.json` 标准题库文件注入到本地部署的 `DTCUP-Quiz` 交互式刷题系统中。
    * 实现脱机的、高频针对性的自动化模拟测试。

4.  **实体资料库生成 (Printable Knowledge Base)**
    * 针对核心 Markdown 笔记，利用 Windows 宿主机原生渲染引擎进行无损排版。
    * 规避 Linux (WSL2) 环境下的中文字体缺失问题，批量导出包含本地相对路径图片的高清 PDF，用于离线背诵与实体打印。

## 💻 技术栈与环境配置
* **开发环境**：Windows 11 + WSL2 (Ubuntu 20.04/22.04) / 云服务器
* **网络工具**：配置全局/终端环境变量代理（Port: 6244/7897），确保 GitHub 资源无缝同步。
* **数据处理**：Python 3.x (Regex, Pandas 数据清洗)
* **版本控制**：Git & GitHub (支持 SSH/Token 认证)
